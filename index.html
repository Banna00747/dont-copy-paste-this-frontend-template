Browserify example

By copy-pasting this project template, you get quickly started in frontend development. Before doing that, please go through the explanation of why all these tools are needed.

Features
  • Provides basic but understandable features which can be fine-tuned later
  • require('x') syntax in your frontend to organize code and use NPM modules
  • New ES6 features via Babel
  • Automatically build bundle.js or bundle.css after changes to code
  • Minify JS and CSS for production
  • Build time configuration for JS
  • LESS to organize CSS code across files plus other neat features
  • Autoprefixer so you can stop worrying about CSS browser prefixes
  • No livereload
Commands
  • npm start Open your browser at http://localhost:8080 and start coding.
  • npm run build To build minified versions of bundle.js and bundle.css for production.
How it works
  • index.html is the entry point to your frontend.
  • styles/ contains all styles. All code in this directory will be built to bundle.css
  • scripts/ contains all JS code. All code in this directory will be built to bundle.js
  • assets/ contains all static assets, such as images.

JavaScript production build

  1. Browserify builds JS code to bundle.js
  2. UglifyJS minifies bundle.js in place

CSS production build

  1. Less builds all styles to bundle.css
  2. PostCSS applies transformations to bundle.css, e.g. Autoprefixer
  3. CleanCSS minifes bundle.css
Summary of tools
  • Browserify to divide JS code across files.
    • Babel transform for Browserify with ES2015 preset to support new ES6 features.
    • Envify transform for Browserify to allow build time configuration.

      As an example, you could build a bundle.js which points to production backend with: API_URL=https://my-app-prod.herokuapp.com npm run build

  • Less.js to divide CSS code across files.
  • clean-css for minifying CSS.
  • UglifyJS for minifying JS.
  • http-server to serve the frontend files at http://localhost:8080.
  • chokidar-cli to watch changes in **/*.less files and rebuild bundle.css in local development.
  • PostCSS to automatically apply certain modifications to CSS.
    • Autoprefixer plugin for PostCSS to automatically add vendor prefixes to CSS. You can write clean CSS without worrying browser prefixes too much.
